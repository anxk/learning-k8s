# k8s 0.3 源码分析

* pkg/api
API 对象定义。

* pkg/apiserver
API Server 实现。

* pkg/capabilities
管理系统的能力级别。

* pkg/client
客户端的实现。

* pkg/cloudprovider

* pkg/constraint
约束条件，用于保障调度一组容器到 Minion 上时不会产生（资源）冲突。

* pkg/controller
ReplicationManager 逻辑，用于控制 ReplicationController 资源对象。

* pkg/conversion
类型转换系统。

* pkg/election
Master 选举逻辑。

* pkg/health
容器健康检查，包含 exec、http 和 tcp 三种类型的探针。

* pkg/healthz
服务检测，主要提供函数 http.HandleFunc handleHealthz，用于 http 服务的健康检测，检测路径为 '/healthz'。

* pkg/httplog

* pkg/kubecfg

* pkg/kubelet

* pkg/labels
提供一个标签管理系统。

* pkg/master

* pkg/proxy

* pkg/registry
资源对象的存储系统。
  * binding
  * controller
  * endpoint
  * etcd
  * minion
  * pod
  * registrytest
  * service

* pkg/runtime

* pkg/scheduler
调度逻辑，主要定义了 Scheduler、MinionLister 和 PodLister 三个接口。另外，还提供了三个调度器，分别是 RandomScheduler、RandomFitScheduler 和 RoundRobinScheduler。

* pkg/service
同步 registry 中的 Service 信息。

* pkg/tools
与 etcd 交互的相关操作。

* pkg/util

* pkg/version
版本信息

* pkg/volume
卷管理逻辑。

* pkg/watch
watchable 对象相关。
